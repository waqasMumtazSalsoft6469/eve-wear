## App Architecture

### Navigation Flow

```
App (Provider, SafeAreaProvider)
└── Routes (auth_token / isFirstTime from Redux)
    ├── Auth Stack   (when not logged in)
    │   ├── Onboarding (initial)
    │   ├── Login
    │   ├── Signup
    │   ├── Forgot
    │   └── Register
    │
    └── Main Stack  (when logged in)
        └── DrawerProvider
            └── AnimatedDrawer (custom side drawer)
                └── Stack.Navigator
                    ├── Tabs (Bottom Tabs) — default
                    │   ├── Home
                    │   ├── CycleOverview
                    │   ├── ChatAi
                    │   ├── Categories
                    │   └── Shop
                    └── Main stack screens (from drawer)
                        ├── MyOrders
                        ├── Subscription
                        ├── MySessions
                        ├── ContactUs
                        ├── OurCollaboration
                        └── Insurance
```

- **Root switch**: `Routes.tsx` uses `isFirstTime` and `auth_token` from Redux to show either **Auth** or **Main**.
- **Main**: Wrapped in `DrawerProvider` and `AnimatedDrawer`. Drawer menu navigates to **Tabs** (with a specific tab) or to **main stack screens** (e.g. My Orders).
- **Routes config**: All screen names live in `src/constants/routes.ts` (`routes.auth`, `routes.main`, `routes.tab`, `routes.navigator`). Use these constants for navigation and registration.

### Key Patterns

- **Screens**: One folder per screen (e.g. `Login/Login.tsx`, `Login/styles.ts`). Exported from `screens/index.ts` and registered in `constants/routes.ts`.
- **Navigation**: Use `routes.main.myOrders`, `routes.tab.home`, etc., and navigate via `navigation.navigate(routes.navigator.main, { screen: 'MyOrders' })` or with tab params when targeting tabs.
- **Drawer**: Open from header with `useDrawerSafe()?.open()`. Navigate from drawer: tab screens go to `Main` → `Tabs` with `params: { screen }`; main-only screens go to `Main` with `screen: screenName`. Navigate first, then close drawer for a smooth transition.
- **Animations**: Use `useFadeSlide`, `useStagger`, `usePressScale` from `@/hooks/animations/` on auth and Home; OnBoard uses `consumeFirstOnBoardAfterLaunch()` so delay runs only after splash, not after logout.
- **Icons / Images**: Run `npm run generate:icons` and `npm run generate:images` after adding SVGs or images; they update `MyIcons.tsx` and `assets/images/index.ts`.

### Boot & Auth Flow

1. **App.tsx**: Renders `SafeAreaProvider` → `Provider` (Redux) → `Routes`. Calls `getLocalItem()` then `BootSplash.hide()`. Can show `AnimatedBootSplash` until `onAnimationEnd`, then unmount.
2. **getLocalItem** (in `utils/checkStorage.ts`): Reads `IS_FIRST_TIME`, language, theme from secure storage and dispatches to Redux; used to show Auth vs Main and restore preferences.
3. **Auth**: Login/Register use Formik + Yup; successful login dispatches `loginAction(userData)`. Logout uses `clearDataAction()`; Routes then switch to Auth stack.
4. **Splash vs OnBoard**: `splashState.consumeFirstOnBoardAfterLaunch()` is used so OnBoard entrance animation is delayed only on first open (after splash), not when returning from logout.

## Setup

### Prerequisites

- Node.js >= 20
- iOS: Xcode, CocoaPods  
- Android: Android Studio, SDK

### Install & Run

```bash
git clone <repo-url>
cd evewear
npm install
# iOS
cd ios && pod install && cd ..
npm run ios
# Android
npm run android
```

### Firebase (optional)

- **Android**: Add `google-services.json` in `android/app/`.
- **iOS**: Add `GoogleService-Info.plist` in `ios/<appname>/`, then `cd ios && pod install`.

### Path alias

`@` points to `src/` (see `babel.config.js` and `tsconfig.json`). Use `@/components/...`, `@/redux/...`, etc.
